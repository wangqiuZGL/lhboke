## Ajax

全称是：Asynchronous JavaScript And XML，翻译过来就是“异步的 Javascript 和 XML”。

- **特点：局部刷新页面，无需重载整个页面。**

```js
//利用XMLHttpRequest模块实现了一个最简单的get网络请求
function ajax(url) {
  const xhr = new XMLHttpRequest();
  //open方法的三个参数：请求方法，请求地址，是否异步；默认是异步操作
  xhr.open("get", url, true);
  xhr.onreadystatechange = function () {
    // 异步回调函数
    if (xhr.readyState === 4) {
      if (xhr.status === 200) {
        console.info("响应结果", JSON.parse(xhr.response).data);
      }
    }
  };
  xhr.send(null);
}
ajax("http://localhost:3000/api/get-data?name=age=20");
```

## Fetch API

Fetch 是在 ES6 出现的，它使用了 ES6 提出的 promise 对象。它是 XMLHttpRequest 的替代品。

> 把它与 Ajax 作比较，其实这是不对的，我们通常所说的 Ajax 是指使用 XMLHttpRequest 实现的 Ajax，所以真正应该和 XMLHttpRequest 作比较。

- **特点：**
  - 使用 promise，不使用回调函数
  - 采用模块化设计，比如 rep、res 等对象分散开来，比较友好。
  - 通过数据流对象处理数据，可以提高网站性能。

```js
<script>
  //采用promise中的.then链式调用的方法处理结果 function Fetch(url){" "}
  {fetch(url)
    .then((res) => res.json())
    .then((data) => {
      console.info(data);
    })}
  Fetch('https://smallpig.site/api/categorygetCategory')
</script>
```

## axios

`Axios`是一个基于`promise`封装的网络请求库，它是基于 XHR 进行二次封装。

- **特点:**
  - 支持 Promise API
  - 拦截请求和响应
  - 转换请求数据和响应数据
  - 自动转换 JSON 数据
  - 客户端支持防御 XSRF
  - 支持 async/await 用法

```js
// 发送 POST 请求
axios({
  method: "post",
  url: "/user/12345",
  data: {
    firstName: "Fred",
    lastName: "Flintstone",
  },
});
```

## 使用 axios 实现请求和响应拦截器

**首先要明确 axios 为什么要进行封装？**
虽然默认直接导入并使用 axios，通过默认的 get、post 方法也能完成数据请求。但是为了避免重复书写请求地址，以及请求前的验证和请求后的操作。通过对 axios 进行封装，来减少代码的容易。

- **目的**
  - 默认配置的统一，如基地址、响应时间、请求头
  - 实现请求拦截
  - 实现响应拦截
  - 常见错误信息处理
  - api 集中式管理

:::tip 💡 封装操作 1.初始化 axios 实例 2.设置请求拦截 3.设置响应拦截 4.集中管理接口
:::

### 1.初始化 axios 实例

主要操作：配置基地址，请求超时，请求头等

```js
const request = axios.create({
  baseURL: "http://pcapi-xiaotuxian-front-devtest.itheima.net",
  timeout: 5000, // 请求超时时间
  headers: {
    "Content-Type": "application/json",
  },
  withCredentials: false, // 跨域请求是否需要携带 cookie
});
```

### 2.设置请求拦截

请求拦截器用于处理请求，并可以在请求发送之前进行一些操作，例如添加认证头或 token 认证，或者取消请求。**请求拦截分为请求成功 config 与请求失败 error**。

```js
request.interceptors.request.use(
  (config) => {
    // 假设 Token 存在 localStorage 中
    const token = localStorage.getItem("token");
    if (token) {
      // 按照后端要求，将Token添加到 Header中(通常是 Authorization)
      config.headers["Authorization"] = `Bearer ${token}`;
    }
    console.log("请求配置", config);
  },
  (error) => {
    Promise.reject(error);
  }
);
```

### 3.设置响应拦截

axios 请求的返回结果里面包含了很多东西，我们的业务层面通常只需要后端返回的数据即可，所以我们需要设置相应拦截，在响应结果返回给业务层之前做一些操作。如统一处理错误码等。响应拦截一般是**接口返回后，统一处理结果（解构数据、处理错误码）**。

```js
request.interceptors.response.use(
  (response) => {
    const res = response.data;
    if (res.code !== 200) {
      ElMessage.error(res.msg || "系统错误");
      // 特殊错误码处理：比如 401 token 失效
      if (res.code === 401) {
        // 清除过期 Token
        localStorage.removeItem("token");
        // 跳转回登录页 (可以使用 Vue Router)
        // window.location.href = '/login'
        ElMessage.warning("登录已过期，请重新登录"); //Element Plus中的消息提示组件
      }
      return Promise.reject(new Error(res.msg || "Error"));
    }
    //返回res或者res.data
    console.log("请求配置", res);
  },
  (error) => {
    // 处理 HTTP 网络错误 (如 404, 500, 网络断开)
    let message = "";
    const status = error.response ? error.response.status : 0;
    switch (status) {
      case 400:
        message = "请求参数错误";
        break;
      case 401:
        message = "未授权，请登录";
        break;
      case 403:
        message = "拒绝访问";
        break;
      case 404:
        message = "请求地址出错";
        break;
      case 408:
        message = "请求超时";
        break;
      case 500:
        message = "服务器内部错误";
        break;
      case 502:
        message = "网关错误";
        break;
      case 503:
        message = "服务不可用";
        break;
      default:
        message = "网络连接故障";
        break;
    }
    ElMessage.error(message);
    return Promise.reject(error);
  }
);

//导出实例
export default request;
```

### 4.集中管理接口

为了方便管理数据请求，减少业务代码量，将数据接口进行集中管理。

```js
import request from "@/utils/request";

// 登录接口
export function login(data) {
  return request({
    url: "/user/login",
    method: "post",
    data, // post 请求体
  });
}

// 获取用户信息接口
export function getUserInfo(id) {
  return request({
    url: "/user/info",
    method: "get",
    params: { id }, // get 参数 (会自动拼接成 url?id=xxx)
  });
}
```
