::: details 背景
ES6 之前，异步编程靠回调函数，多层嵌套会形成 “回调地狱”。回调地狱的问题是代码可读性差和错误处理困难，比如下面面对每一个环节如果出现问题，要给出对应的"登录失败""获得用户ID失败"等等
**场景**: 一个真实的业务场景 - 用户登录后获取个人信息
假设我们要完成以下步骤：
1. 用户登录，获取 token
2. 用token 获取用户 ID
3. 用用户 ID 获取用户详细信息
4. 用用户信息获取用户的订单列表
5. 用订单 ID 获取订单详情
```js
// 回调地狱：嵌套越深，可读性越差
login('zhangsan', '123456', (loginResult) => {
  getUserId(loginResult.token, (userIdResult) => {
    getUserInfo(userIdResult.userId, (userInfo) => {
      console.log('用户信息:', userInfo);
      getOrders(userIdResult.userId, (orders) => {
        console.log('订单列表:', orders);
        getOrderDetail(orders[0].orderId, (orderDetail) => {
          console.log('订单详情:', orderDetail);
          // 😱 如果还有更多步骤，嵌套会更深...
        });
      });
    });
  });
});
```
:::

## 原理部分
为了解决回调地狱的问题，就先后出现了promise、Generator和async/await。
### 1.Promise的解决思路
Promise 是一个代表异步操作最终完成或失败的对象。
- 将异步操作的结果封装为一个「状态机」（pending/fulfilled/rejected），状态一旦改变不可逆转；
- 用`.then()`链式调用替代嵌套回调，异步流程线性化；
- 用`.catch()`统一捕获错误，替代每层回调的 err 判断。

```js
//1.将异步操作封装为一个对象
function login(username, password) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      console.log('1. 登录成功');
      resolve({ token: 'abc123' });
    }, 1000);
  });
}


//2.使用链式调用 - 扁平化
login('zhangsan', '123456')
  .then(loginResult => {
    return getUserId(loginResult.token);
  })
  .then(userIdResult => {
    return getUserInfo(userIdResult.userId);
  })
  .then(userInfo => {
    console.log('用户信息:', userInfo);
    return getOrders(userInfo.userId || '001');
  })
  .then(orders => {
    console.log('订单列表:', orders);
    return getOrderDetail(orders[0].orderId);
  })
  .then(orderDetail => {
    console.log('订单详情:', orderDetail);
  })
  .catch(error => {
    // 统一的错误处理！
    console.error('发生错误:', error);
  });
```
promise在通过链式调用，处理回调函数使得代码的可阅读性更好，统一的错误处理也能减少代码量等。但是Promise也存在一定的缺陷：
1.任然存在一定的嵌套;
2.因为promise的特性导致无法中断promise链
3.错误处理不够直观，排查错误困难

### 2.Generator 的过渡
Generator是ES6引入的一种特殊函数，可以暂停执行和恢复执行。
**核心思想**:
- 函数可以"暂停"和"继续"
- 用同步的方式写异步代码
- 需要手动控制执行流程


::: code-group

```javascript [手动执行]

function* fetchUserData() {
  console.log('开始获取数据');
  
  // 看起来像同步代码！
  const loginResult = yield login('zhangsan', '123456');
  console.log('1. 登录成功:', loginResult);
  
  const userIdResult = yield getUserId(loginResult.token);
  console.log('2. 获取用户ID:', userIdResult);
  
  const userInfo = yield getUserInfo(userIdResult.userId);
  console.log('3. 用户信息:', userInfo);
  
  const orders = yield getOrders(userIdResult.userId);
  console.log('4. 订单列表:', orders);
  
  const orderDetail = yield getOrderDetail(orders[0].orderId);
  console.log('5. 订单详情:', orderDetail);
  
  return orderDetail;
}

// 手动执行 Generator（需要一个执行器）
function runGenerator(generatorFunc) {
  const gen = generatorFunc();
  
  function step(value) {
    const result = gen.next(value);
    
    if (result.done) {
      return result.value;
    }
    
    // 如果 yield 的是 Promise，等待它完成
    if (result.value && typeof result.value.then === 'function') {
      result.value
        .then(val => step(val))
        .catch(err => gen.throw(err));
    } else {
      step(result.value);
    }
  }
  step();
}
// 执行
runGenerator(fetchUserData);
```

### 3.async/await
async/await 做的事情就是将 Generator 函数转换成 Promise，说白了，async 函数就是 Generator 函数的语法糖，await 命令就是内部 then 命令的语法糖。async 函数就是将 Generator 函数的星号（*）替换成 async，将 yield 替换成 await，仅此而已。其实 async 函数的实现原理，就是将 Generator 函数和自动执行器(比如 co)，包装在一个函数里。

**核心思想**:
- 用同步的方式写异步代码
- 基于 Promise，但更简洁
- 原生支持，无需额外库

```js
//通过同步的方式，实现回调和异步
async function fetchData() {
  try {
    const user = await getUser();
    const orders = await getOrders(user.id);
    const detail = await getOrderDetail(orders[0].id);
    console.log(detail);
  } catch (error) {
    console.error(error);
  }
}
```

## 手写Promise
上面讲了从回调地狱到promise再到async/await。现在专门针对promise进行深入理解。
**Promise的状态**: `pending`、`fulfilled`(resolve后的状态)、`rejected`(reject后的状态)
**两个特点**：
  - 对象的状态不受外界影响;
  - 一旦状态改变，就不会再变，任何时候都可以得到这个结果(状态不可逆)。Promise 对象的状态改变，只有两种可能：从 pending 变为 fulfilled 和从 pending 变为 rejected。只要这两种情况发生，状态就凝固了，不会再变了，会一直保持这个结果，这时就称为 resolved（已定型）。

实例方法 - `.catch(...)`,`.finally(...)`
静态方法 - `resolve(...)`,`reject(...)`,`race(...)`,`all(...)`,`allSettled(...)`,`any(...)`


::: details 💡 Promise
手写实现一个好的promise要如何一步步实现
手写 Promise 主要分为六个步骤：

**第一步，实现基础框架**。定义三个状态常量，在构造函数中初始化状态、
值和原因，定义 resolve 和 reject 函数来改变状态，最后执行 executor。

**第二步，实现then方法**。做参数校验实现值穿透，根据状态执行对应的回调。

**第三步，实现链式调用**。then 方法返回一个新的 Promise，在新 Promise 中
执行回调并 resolve 返回值。

**第四步，处理异步**。定义回调队列，如果状态是 pending，将回调存入队列，
等异步完成后再执行。

**第五步，实现Promise解析过程**。用 resolvePromise函数处理then回调的
返回值，支持返回 Promise、thenable对象和普通值。

**第六步，实现静态方法**。包括 resolve、reject、all、race、allSettled、any 等。

核心难点是 resolvePromise 函数和异步处理的回调队列。
::: 

### 1.实现基础框架
定义三个状态常量，在构造函数中初始化状态、值和原因，定义 resolve 和 reject 函数来改变状态，最后执行 executor。执行executor，目的是在new Promise((resolve, reject) => { ... })内的同步代码有错误时，抛出错误
```js
定义三个状态常量，在构造函数中初始化状态、值和原因，定义 resolve 和 reject 函数来改变状态，最后执行executor。
const PADDING = "padding";
const FULFILLED = "fulfilled";
const REJECTED = "rejected";
class myPromise {
  constructor(executor) {
    //初始化状态
    this.status = PADDING;
    this.result = undefined;
    this.reason = undefined;
    //定义两个箭头函数，能接收实例的参数
    const resolve = (result) => {
      //改变状态
      if (this.status === PADDING) {
        this.status = FULFILLED;
        this.result = result
      }
    };
    const reject = (reason) => {
      if (this.status === PADDING) {
        this.status = REJECTED;
        this.reason = reason
      }
    };
    //执行executor，目的是在new Promise((resolve, reject) => { ... })内的同步代码有错误时，抛出错误
    try {
      executor(resolve, reject);
    } catch (error) {
      reject(error);
    }
  }
}
```
### 2.实现then方法
做参数校验实现值穿透，根据状态执行对应的回调。这里的参数校验的目的是判断返回的是否是函数，如果不是函数将值传下去，也就是说`x => x`。如果传入的不是函数，上一个 Promise 的结果（成功值/失败原因）会原封不动传递给下一个then。
```js
const PADDING = "padding";
const FULFILLED = "fulfilled";
const REJECTED = "rejected";
class myPromise {
  constructor(executor){...}
  //then方法中要做什么？在promise中我们通过then进行执行对应的回调
  then(onFulfilled,onRejected) {
    //先做一个是否为函数的判断，如果是函数，接收函数;如果是非函数，进行透值传递，将上一个 Promise 的结果原封不动传递给下一个 then
    onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : x => x;
    onRejected = typeof onRejected === 'function' ? onRejected : x=>{throw x}
    //根据状态进行判断，如果状态是fulfilled，执行onFulfilled传回的函数;状态是rejected,执行onRejected传回的函数
    if (this.status === FULFILLED){
      onFulfilled(this.result)   //这里为什么用this？在构造函数中的this是指定当前实例对象，这里的this因为then方法是被实例对象调用的，所以也是指向实例对象
    } else if(this.status === REJECTED){
      onRejected(this.reason)
    }
  }  
}
```

### 3.回调队列和异步任务
实现then方法后现有的myPromise以及执行对应状态的回调,但是我们不能让其进行多次调用和异步任务。
::: details 问题分析
比如下面的异步调用和多回调任务的场景,我们发现一个问题，因为构建函数内的executor是异步操作，所以会立即执行p.then，但是当前的状态还是padding,当前的代码没有把then的回调存入队列，导致等异步任务结束后，虽然状态修改了，但是已经没有回调可以执行。

![Promise异步](/qian/promise-async.png)

```js [异步操作多次调用场景]
const p = new myPromise((resolve, reject) => {
    setTimeout(() => resolve("异步成功"), 1000);
});

p.then(
  (res) => {
    console.log("then1:", res);
  },
  (err) => {
    console.log("then1:", err);
  }
);

p.then(
  (res) => {
    console.log("then2:", res);
  },
  (err) => {
    console.log("then2:", err);
  }
);
```
:::

总结上面的问题就是--**缺少回调队列 + 异步回调未缓存**。
- 实现异步，then方法中加入异步操作
- 将回调加入到对应的回调队列中
- 在resolve和reject中，将存储的回调队列进行遍历


::: code-group
```js [异步封装]
//也可以用setTimeout
function runAsynctask(callback) {
  if (typeof queueMicrotask === "function") {
    queueMicrotask(callback);
  } else if (typeof MutationObserver === "function") {
    const observer = new MutationObserver(callback);
    const divNode = document.createElement("div");
    observer.observe(divNode, { childList: true });
    divNode.innerText = "监听对象修改了，微任务形式执行回调";
  } else {
    setTimeout(callback, 0);
  }
}
```

```js [then中添加异步和回调队列]
const PADDING = "padding";
const FULFILLED = "fulfilled";
const REJECTED = "rejected";
class myPromise {
constructor(executor) {
  //初始化状态
  this.status = PADDING;
  this.result = undefined;
  this.reason = undefined;
  this.onFulfilledCallbacks = [];
  this.onRejectedCallbacks = [];
  //定义两个箭头函数，能接收实例的参数
  const resolve = (result) => {
    //改变状态
    if (this.status === PADDING) {
      this.status = FULFILLED;
      this.result = result;
      this.onFulfilledCallbacks.forEach(fn => fn());  //这一步的操作是对异步队列中的任务拿出来执行
    }
  };
  const reject = (reason) => {
    if (this.status === PADDING) {
      this.status = REJECTED;
      this.reason = reason;
      this.onRejectedCallbacks.forEach(fn => fn());
    }
  };
  //执行executor，目的是在new Promise((resolve, reject) => { ... })内的同步代码有错误时，抛出错误
  try {
    executor(resolve, reject);
  } catch (error) {
    reject(error);
  }
}

then(onFulfilled, onRejected) {
  //先做一个是否为函数的判断，如果是函数，接收函数;如果是非函数，进行透值传递，将上一个 Promise 的结果原封不动传递给下一个 then
  onFulfilled =
    typeof onFulfilled === "function" ? onFulfilled : (x) => x;
  onRejected =
    typeof onRejected === "function"
      ? onRejected
      : (x) => {
          throw x;
        };
  //根据状态进行判断，如果状态是fulfilled，执行onFulfilled传回的函数;状态是rejected,执行onRejected传回的函数
  if (this.status === FULFILLED) {
    runAsynctask(() => {   //将回调修改为异步任务
      onFulfilled(this.result);
    });
    //这里为什么用this？在构造函数中的this是指定当前实例对象，这里的this因为then方法是被实例对象调用的，所以也是指向实例对象
  } else if (this.status === REJECTED) {
    runAsynctask(() => {   //将回调修改为异步任务
      onRejected(this.reason);
    });
  } else if (this.status === PADDING) {
    //在状态为PADDING时，将回调修改为异步任务，并放到对应的数组(队列)中
    this.onFulfilledCallbacks.push(() => {
      runAsynctask(() => {  
        onFulfilled(this.result);
      });
    }),
      this.onRejectedCallbacks.push(() => {
        runAsynctask(() => {
          onRejected(this.reason);
        });
      });
  }
}
}
```
:::


### 4.实现链式调用
在实现异步和回调队列后，如果想要执行多个异步任务，只能进行嵌套着写。
::: details 💡思考
在进行链式调用这里，为什么要使用`const x = onFulfilled(this.result)`来接收`onFulfilled(this.result)`?那么`onFulfilled(this.result)`又是什么？这就要回到实现then方法中，当时的代码只是简单定义根据不同的状态执行`onFulfilled`还是`onRejected`;还有就是判断是否是函数和值穿透。当时我们要实现的功能就是**根据状态执行对应的函数**。简单的事是根据打印内容，常用的可能是身份验证成功，请求订单详情。
之后我们要实现异步操作和多次调用。所以通过将`onFulfilled`和`onRejected`用异步操作包裹,因为异步操作的状态变化要等实例中是执行`resolve`或`resolve`才变化，所以通过push方法，先将要执行的回调`onFulfilled`和`onRejected`放到对应的数组中,当异步任务结束后开始执行`resolve`或`resolve`,将对应数组内的回调函数进行执行即可。
通过上面的操作，then能处理多次回调函数，但是如果面对下一个回调函数要使用上一个回调函数的数据的场景，还是只能依赖于嵌套。因为then方法缺少能接收上一次回调的结果，以及状态。就像下面的嵌入调用一样，我们要返回的是一个Promise对象，里面要有上一个Promise的状态和返回值。 
```js
const p1 = new myPromise((resolve)=>{
  resolve(1)
})

p1.then(value=>{
    console.log('✅ 第一个 then:', value);
    return value + 1;
  }
)
//等价于什么？
p1.then(res1 => {
  console.log(res1); // 1秒后打印用户信息
  const p2 = new myPromise((resolve) => {  
    setTimeout(() => resolve("用户订单"), 1000); 
  })
})
```
:::

```js
//如先请求用户信息 → 再请求用户订单 → 再请求订单详情
const p1 = new myPromise((resolve) => {
  setTimeout(() => resolve("用户信息"), 1000);
});

// 无链式调用：只能嵌套 then，层级越深越难维护
p1.then(res1 => {
  console.log(res1); // 1秒后打印用户信息
  const p2 = new myPromise((resolve) => {  
    setTimeout(() => resolve("用户订单"), 1000); 
  }); //异步任务中返回根据用户信息查到的用户订单，用p2进行接收
  p2.then(res2 => {
    console.log(res2); // 2秒后打印用户订单
    const p3 = new myPromise((resolve) => {
      setTimeout(() => resolve("订单详情"), 1000);
    });
    p3.then(res3 => {
      console.log(res3); // 3秒后打印订单详情
    });
  });
});
```
从上面的例子中我们能看到then方法中要有新的promise实例，这也是实现**链式调用的核心逻辑--then方法返回新的myPromise实例**。还有就是统一错误。那么是不是想下面的代码将then中回调函数状态判断放到里面即可？显然是不行的，因为当前的回调函数处理中，没办法捕捉上一个Promise的返回值，让新 Promise 状态关联返回值。
```js
const promise2 =new myPromise(()=>{
  if (this.status === FULFILLED){}
  else if (this.status === REJECTED){}
  else if (this.status === PADDING){}
})
----------------------------------------------------------
//修改内容,新 Promise 的状态由上一个 then 回调的结果决定
const promise2 =new myPromise(()=>{
if (this.status === FULFILLED) {
  runAsynctask(() => {
    try {
      const x = onFulfilled(this.result);
      resolve(x);
    } catch (error) {
      reject(error);
    }
  });
}
else if (this.status === REJECTED) {
  runAsynctask(() => {
    try {
      const x = onRejected(this.reason);
      resolve(x);  //onRejected 的目标不是把错误继续向下传，而是让你有机会把错误转为正常值并继续链式调用。
    } catch (error) {
      error;
    }
  });
}
//else if (this.status === PADDING){}同理
})
return promise2;
//以及加入
catch(onRejected) {
  return this.then(null, onRejected);
}
```
::: warning
**为什么在onRejected中也是执行resolve(x)?**
onRejected的返回值意味着“我已经处理完错误，并给下一个then一个新值”，所以必须 resolve(x) 而不是 reject(x)。如果返回的是状态是REJECTED，那么就无法进行下一个then的执行。
:::

### 5.实现Promise解析过程
用resolvePromise函数处理then回调的返回值，支持返回 Promise、thenable对象和普通值。
resolvePromise是Promise A+ 规范中最关键的部分，它决定x应该如何影响promise2。。我们要考虑promise的边界，现在并不知道返回的x是什么？
```js
1. 普通值             → resolve(value)
2. Promise           → 必须跟随其状态
3. thenable 对象     → 必须尝试调用它的 then
4. 出错的 getter     → 必须 reject
5. 自己返回自己       → 必须报循环引用  
6. then 调用了多次    → 必须忽略第二次
```

它要处理四种情况：

1. 如果返回值和 promise2 是同一个对象，抛出循环引用错误
2. 如果返回值是 Promise，采用它的状态
3. 如果返回值是 thenable 对象（有 then 方法），当作 Promise 处理
4. 如果是普通值，直接 resolve
```js
function resolvePromise(promise2, x, resolve, reject) {
  // 1. 如果 promise2 和 x 是同一个对象，抛出错误（防止循环引用）
  if (promise2 === x) {
    return reject(new TypeError('Chaining cycle detected'));
  }
  
  // 2. 如果 x 是 Promise，采用它的状态
  if (x instanceof myPromise) {
    x.then(resolve, reject);
    return;
  }
  
  // 3. 如果 x 是对象或函数
  if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
    let called = false; // 防止多次调用
    
    try {
      // 取出 x.then
      const then = x.then;
      
      // 如果 then 是函数，说明 x 是 thenable 对象
      if (typeof then === 'function') {
        //then.call(x,resolve,reject),在resolve加一段判断，不能重复执行resolve;如果执行reject，直接就返回reject(r)
        then.call(
          x,
          // 成功回调
          y => {
            if (called) return;
            called = true;
            // 递归解析（y 可能还是 Promise）
            resolvePromise(promise2, y, resolve, reject);
          },
          // 失败回调
          r => {
            if (called) return;
            called = true;
            reject(r);
          }
        );
      } else {
        // then 不是函数，直接 resolve
        resolve(x);
      }
    } catch (error) {
      if (called) return;
      called = true;
      reject(error);
    }
  } else {
    // 4. 如果 x 是普通值，直接 resolve
    resolve(x);
  }
}
```
在函数内使用是
```js
try {
  const x = onFulfilled(this.value);
  // 使用 resolvePromise 解析返回值
  resolvePromise(promise2, x, resolve, reject);
} catch (error) {
  reject(error);}
```

```js
//测试
const p1 = new myPromise((resolve, reject) => {
  resolve(1);
});
p1.then((value) => {
  return new myPromise((resolve) => {
    resolve(value + 1);
  });
}).then((value) => {
  console.log(value); // 2 ✅
});

```

## Promise静态方法
###  手写Promise的.catch方法
catch是Promise的实例方法，用来处理拒绝（reject）情况的简化写法。也就是说，如果 Promise 最终是 reject，就会执行 onRejected。
```js
catch(onRejected) {
  return this.then(null, onRejected);
}

//等价于
//promise.then(undefined, onRejected)
```

### finally(callback)
无论Promise是成功还是失败，都执行callback的方法。
- 不关心 Promise 的结果
- `callback()` 的返回值不会影响链式调用的结果
- 会把前一个 Promise 的值继续向下传递

```js
finally(callback) {
  return this.then(
    value => MyPromise.resolve(callback()).then(() => value),
    reason => MyPromise.resolve(callback()).then(() => { throw reason })
  );
}
```

### static resolve(value)
**把任意值包装成一个`myPromise`，并且处于`fulfilled`状态。**

::: code-group
```js [写法]
static resolve(value) {
  // 如果是 Promise，直接返回
  if (value instanceof MyPromise) {
    return value;
  }
  // 否则返回一个新的 fulfilled Promise
  return new MyPromise((resolve) => {
    resolve(value);
  });
}
```
```js [使用示例]
MyPromise.resolve(123).then(console.log); // 123
```
:::

### static all(promises)
接收一个数组，**所有Promise成功，才成功**；只要有一个失败，就立即失败。
::: code-group
```js [写法]
static all(promises) {
  return new MyPromise((resolve, reject) => {
    // 参数校验
    if (!Array.isArray(promises)) {
      return reject(new TypeError('Argument must be an array'));
    }
    
    const result = [];
    let count = 0;
    const len = promises.length;
    
    // 空数组直接返回
    if (len === 0) {
      return resolve(result);
    }
    
    promises.forEach((promise, index) => {
      // 确保是 Promise
      MyPromise.resolve(promise).then(
        value => {
          result[index] = value;
          count++;
          // 所有 Promise 都成功才 resolve
          if (count === len) {
            resolve(result);
          }
        },
        reason => {
          // 有一个失败就 reject
          reject(reason);
        }
      );
    });
  });
}
```
```js [使用示例]
MyPromise.all([
  MyPromise.resolve(1),
  2,
  MyPromise.resolve(3)
]).then(console.log); // [1, 2, 3]
```
:::

### static race(promises)
**谁先结果（成功/失败），就采用谁的结果**
::: code-group
```js [写法]
static race(promises) {
  return new MyPromise((resolve, reject) => {
    if (!Array.isArray(promises)) {
      return reject(new TypeError('Argument must be an array'));
    }
    
    // 第一个完成的 Promise 决定结果
    promises.forEach(promise => {
      MyPromise.resolve(promise).then(resolve, reject);
    });
  });
}
```
```js [使用示例]
//第一个完成的 Promise 决定结果
MyPromise.race([
  MyPromise.resolve(1),
  new MyPromise(res => setTimeout(() => res(2), 1000))
]).then(console.log); // 1
```

### static allSettled(promises)
- 无论成功或失败，都会返回一个数组，该数组的成员顺序与传入的 Promise 对象的顺序保持一致。
- 传入的元素是 Promise，返回的数组元素对象的就是那个 Promise 成功/失败的值
- 传入的元素不是 Promise，返回的数组元素对象的 status 为 fulfilled，value 为传入的元素值（具体看下面测试的例子结果）

::: code-group
```js [写法]
static allSettled(params) {
  return new MyPromise((resolve, reject) => {
    const arr = []
    let index = 0
    // 判断是否所有的promise都执行完毕
    const isPromiseAllEnd = (key, value) => {
      arr[key] = value
      index++
      if (index === params.length) {
        resolve(arr)
      }
    }
    for (let i = 0; i < params.length; i++) {
      let current = params[i]
      if (isPromise(current)) {
        current.then(
          (res) => {
            isPromiseAllEnd(i, {
              status: 'fulfilled',
              value: res
            })
          },
          (err) => {
            isPromiseAllEnd(i, {
              status: 'rejected',
              reason: err
            })
          }
        )
      } else {
        isPromiseAllEnd(i, {
          status: 'fulfilled',
          value: current
        })
      }
    }
  })
}
```
```js [使用示例]
MyPromise.allSettled([
  Promise.resolve('p1'),
  Promise.reject('p2'),
  Promise.resolve('p3'),
  4,
  5,
  new Error('error')
]).then((results) => {
  console.log('success: ', results)
})

/** 返回的内容
 * success:  success:  [
  { status: 'fulfilled', value: 'p1' },
  { status: 'rejected', reason: 'p2' },
  { status: 'fulfilled', value: 'p3' },
  { status: 'fulfilled', value: 4 },
  { status: 'fulfilled', value: 5 },
  {
    status: 'fulfilled',
    value: Error: error
  }
]
 */
```

### static any(promises)
只需要有一个 Promise 成功就resolve。如果全部失败，就 reject，并返回一个 AggregateError（包含所有错误）

::: code-group
```js [写法]
static any(promises) {
  return new MyPromise((resolve, reject) => {
    if (!Array.isArray(promises)) {
      return reject(new TypeError('Argument must be an array'));
    }
    
    const errors = [];
    let count = 0;
    const len = promises.length;
    
    if (len === 0) {
      return reject(new AggregateError('All promises were rejected'));
    }
    
    promises.forEach((promise, index) => {
      MyPromise.resolve(promise).then(
        value => {
          // 有一个成功就 resolve
          resolve(value);
        },
        reason => {
          errors[index] = reason;
          count++;
          // 所有都失败才 reject
          if (count === len) {
            reject(new AggregateError(errors));
          }
        }
      );
    });
  });
}
```
```js [成功使用示例]
//一个完成成功就resolve
MyPromise.any([
  MyPromise.reject("a"),
  MyPromise.resolve("成功"),
  MyPromise.reject("b")
]).then(console.log); // "成功"
```
```js [全部失败]
//全部失败就 reject
MyPromise.any([
  MyPromise.reject("a"),
  MyPromise.reject("b")
]).catch(err => console.log(err.errors));
```
